# MEMORY.md - 证券产业新闻情报 Agent 长期记忆

## 默认能力

- 支持 WorkBuddy 对话式生成日报。
- 用户可直接说：“基于人设文件和技能文件为我生成今日的某某板块日报”。
- 输出 HTML 形式默认参考：`outputs/securities-news-digest-2026-06-25-p1-review.html`。

## 板块范围

- 具体板块由用户指定或由 `config/topics.json` / `sector_configs` 配置。
- PCB、存储芯片是历史示例板块，不是固定限制。
- 可覆盖机器人、医药、消费电子、新能源车、低空经济、半导体设备、先进封装、AI 算力、服务器、HBM、DRAM、NAND、覆铜板、玻纤布、ABF 载板等证券产业板块。

## 默认运行时间

- 对话式场景按需运行。
- 自动化场景可每日 18:30 固定运行一次。
- 默认覆盖窗口为滚动 24 小时，而不是自然日 00:00 至当前时间；中午生成日报时也必须回看昨晚、盘后和海外夜间信息。

## 输出约定

- HTML 日报：`outputs/securities-news-digest-YYYY-MM-DD.html`
- JSON 记忆：`website/data/daily-YYYY-MM-DD.json`
- 每日运行摘要：`memory/YYYY-MM-DD.md`
- 如果同一天多板块重复生成，文件名可加板块 slug。

## 长期规则

- P0 必须稀缺，优先一手来源或多源交叉验证；P0 候选优先来自滚动 24 小时窗口，早于 24 小时的事件只有在窗口内出现新增验证才可进入 P0 候选。
- 雪球、股吧、论坛、自媒体只能作为弱信号来源，不可单独支撑 P0。
- 日报面向证券行业个人用户，仅提供信息参考，不构成投资建议。
- P0 必须包含事实、影响链条、趋势方向、评分解释、后续跟踪和来源。
- P1 必须包含观察信号卡片、当前强度、升级条件和判断依据。
