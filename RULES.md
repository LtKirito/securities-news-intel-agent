# RULES.md - 证券新闻情报 Agent 规则

## 运行规则

- 支持 WorkBuddy 对话式按需生成日报。
- 用户说“基于人设文件和技能文件生成今日某某板块日报”时，按技能流水线执行。
- 自动化场景可默认每日 18:30 运行一次。
- 只抓取公开网页内容。
- 具体板块由用户指定或由 `config/topics.json` / `sector_configs` 配置。
- PCB、存储芯片只是历史示例板块，不是固定限制。
- 每次运行必须生成 HTML 和 JSON 两份结果，建议同时写运行摘要。
- 默认覆盖窗口为滚动 24 小时，而不是自然日截至当前时间；中午生成日报时也必须回看过去 24 小时，避免漏掉昨晚、盘后和海外夜间发布的强信号。

## 来源规则

来源分级：

| 等级 | 来源类型 | 说明 |
|---|---|---|
| S | 公司公告、交易所公告、企业官网、投资者关系活动记录 | 一手来源，可信度最高 |
| A | 东方财富、新浪财经、同花顺、证券时报、中国证券报、上海证券报、财联社公开网页 | 主流财经来源 |
| B | 行业媒体、产业研究机构、研报摘要聚合页面 | 可参考，但需注意二次转述 |
| C | 雪球、股吧、论坛、社交平台公开内容 | 弱信号，只能作为辅助 |
| X | 无来源、自媒体搬运、标题党、旧闻翻炒 | 默认噪音 |

## 评级规则

每条候选新闻按五个维度打分，每项 1-5 分：

1. 板块影响：是否可能影响用户指定板块的整体预期。
2. 产业链相关性：是否直接涉及材料、设备、制造、封测、终端需求、价格、订单、产能。
3. 可信度：是否来自一手来源、主流财经源或多源交叉验证。
4. 时效性：是否处于滚动 24 小时窗口内，或虽早于 24 小时但在窗口内出现一手公告/权威媒体确认/订单产能价格客户等新增验证；是否不是旧闻翻炒。
5. 趋势价值：是否能改变后续跟踪方向或形成连续信号。

评级标准：

| 等级 | 标准 | 处理 |
|---|---|---|
| P0 | 板块影响 >=4，可信度 >=4，趋势价值 >=4，且具备明确边际变化 | 放入日报头部，必须写事实、影响链条、趋势方向、评分解释、后续跟踪点 |
| P1 | 总分 >=16，或产业链相关性 >=4 且可信度 >=3 | 放入重点新闻，必须写观察信号卡片和升级条件 |
| P2 | 有参考价值但影响有限、验证不足或短期边际变化较弱 | 放入一般关注 |
| Noise | 可信度低、重复转载、标题党、旧闻翻炒、无实质信息 | 过滤，并记录原因 |

## P0 收紧规则

P0 候选的时间窗口规则：

- 优先纳入滚动 24 小时内的新信息。
- 早于 24 小时的事件，只有在滚动 24 小时内出现一手公告、权威媒体确认、订单/合同、产能、价格、客户、政策细则、财报/业绩等新增验证时，才可进入 P0 候选。
- 24 小时内的信息也不能自动评为 P0，仍必须满足高可信、强产业边际变化和趋势价值。

以下情况禁止直接评为 P0：

- 只有雪球、股吧、论坛或单一自媒体来源。
- 只有股价异动，没有事实原因。
- 只是机构观点重复，没有新增事实。
- 只是概念炒作，没有公司、订单、价格、产能、政策或需求侧证据。
- 来源正文无法访问或没有原始链接。

## 日报结构

HTML 日报必须参考 `outputs/securities-news-digest-2026-06-25-p1-review.html` 的形式，包含：

1. 今日核心结论
2. P0 关键新闻
3. P1 重点新闻
4. P2 · 一般关注
5. 趋势判断
6. 产业链共振点
7. 分歧点
8. 噪音过滤
9. 后续跟踪清单
10. 信息来源
11. 免责声明

## 写作规则

- 中文输出。
- 先结论，后细节。
- 事实和判断分开。
- 每条新闻必须带来源链接。
- 每条新闻必须标注评级、情绪、可信度、是否验证。
- P0 必须展示影响链条、趋势方向、评分解释和后续跟踪。
- P1 必须展示观察信号卡片、当前强度和升级条件。
- 不写空泛废话，不重复标题。

## 禁止事项

- 禁止具体买卖建议。
- 禁止目标价预测。
- 禁止承诺收益。
- 禁止仓位建议。
- 禁止把未经验证传闻写成事实。
- 禁止把中国香港、中国台湾、中国澳门列为国家。
