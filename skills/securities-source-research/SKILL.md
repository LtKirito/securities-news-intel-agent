# Securities Source Research

## 职责

只负责公开来源采集，输出候选新闻。不得做最终 P0/P1/P2 评级，不写趋势结论，不写日报正文。

适用于任意证券产业板块，例如机器人、医药、消费电子、新能源车、低空经济、半导体、PCB、存储芯片等。具体板块来自用户对话或 `sector_configs`。

## 对话模式输入

用户通常会说：

- “生成今日机器人板块日报”
- “基于人设和技能文件生成今天半导体设备日报”
- “帮我做一份低空经济证券产业日报”

从用户话语中提取：

- `sector_name`：板块名称。
- `date`：默认今日。
- `date_window`：默认滚动 24 小时，即当前生成时间向前回看 24 小时；不要默认使用自然日 00:00 至当前时间，除非用户明确指定。
- `reference_report_style`：如用户附带 HTML 参考日报，记录其路径供报告阶段使用。

## 采集策略

### 搜索词生成

围绕目标板块生成 8-15 组搜索词：

- 板块名 + “产业链” / “订单” / “涨价” / “供需” / “产能” / “政策” / “业绩” / “公告”。
- 板块核心公司 + “公告” / “投资者关系” / “订单” / “扩产” / “业绩预告”。
- 上游材料、设备、制造、终端需求关键词。
- 昨日跟踪项中的验证指标。

不要只搜股价异动。日报关注产业事实和边际变化。搜索时必须覆盖滚动 24 小时窗口，尤其包括前一日晚间、盘后公告和海外夜间信息。

### 多轮召回规则

对话式生成日报时，采集质量决定日报上限，必须主动扩大召回：

- 第一轮：板块名 + 产业事实关键词，覆盖公告、订单、价格、政策、产能、供需、业绩。
- 第二轮：核心公司、上游材料/设备、下游终端需求、历史跟踪项验证指标。
- 第三轮：如果候选新闻少于 8 条，扩大到同义词、细分赛道、产业链上下游和主流财经站点限定搜索。
- 每个板块目标候选量为 10-25 条；少于 5 条时必须标记 `coverage_warning`。
- 至少尝试覆盖 1 类一手来源和 2 类主流财经来源；如果没有找到，写入 `source_gap`。
- 不为了凑数量引入明显无关、旧闻翻炒或纯行情噪音。

### 来源优先级

优先：

- S 级：公司公告、交易所公告、企业官网、投资者关系记录。
- A 级：东方财富、新浪财经、同花顺、证券时报、中国证券报、上海证券报、财联社公开网页。
- B 级：行业媒体、产业研究机构、研报摘要聚合页面。
- C 级：雪球、股吧、论坛、社交平台公开内容，只作为弱线索。

### URL 安全规则

用户手动链接和搜索结果都要过滤：

- 只允许 `http://` 和 `https://`。
- 禁止 `file://`、`ftp://`、`data:`、`javascript:`。
- 禁止 localhost、127.0.0.0/8、0.0.0.0、内网网段、链路本地地址、云元数据地址。
- 禁止含用户名密码的 URL。
- 重定向后仍需重新校验。
- 不绕过登录、付费墙、内网或访问限制。

## 候选新闻字段

每条候选保留：

- `title`
- `summary`
- `published_at`
- `source_name`
- `source_tier`
- `url`
- `matched_sector`
- `matched_keywords`
- `related_companies`
- `is_primary_source`
- `collection_mode`
- `access_issue`
- `raw_text_excerpt`

正文不可访问时不要编造，只保留可见标题/摘要/URL，并写明 `access_issue`。

## 对话模式输出

输出给下一阶段的结构化数据：

```json
{
  "date": "YYYY-MM-DD",
  "date_window": "覆盖窗口",
  "sectors": ["板块名称"],
  "candidates": [],
  "source_counts": [],
  "coverage_warning": "候选不足时填写，否则为空",
  "source_gap": "一手来源或主流财经来源缺失时填写，否则为空"
}
```

可以保存为：`outputs/artifacts/candidates-YYYY-MM-DD.json`，但对话模式不是必须。

## Runner 模式

- 输入 artifact：`workflow_plan.json`
- 输出 artifact：`candidates.json`
- Schema：`schemas/candidates.schema.json`
- 保存路径：`data/users/{user_id}/reports/{date}/run/artifacts/candidates.json`

## 禁止事项

- 不做最终评级。
- 不写趋势结论。
- 不输出买卖建议、目标价、收益承诺、仓位建议。
- 不把社区热帖、论坛讨论直接当作强事实。
- 不采集非公开或受限内容。
